## 2. 적용한 작업 범위

AI가 **초안 작성 · 수정 · 명령 실행**에 관여한 범위이며, 최종적으로는 로컬에서 lint / format / test / build로 검증했습니다.

- **프로젝트 초기 설정**: Vite + React 18 + TypeScript + pnpm, ESLint/Prettier, Tailwind 4, FSD 폴더 구조
- **라우팅·레이아웃**: `react-router-dom`, GNB/LNB 골격
- **데이터·인증**: TanStack Query, `fetch` 기반 HTTP 클라이언트, 401 시 refresh 재시도 등
- **API 타입 정의**: openapi에 맞춘 type 정의
- **MSW**: `public` worker, fixture·handler, 개발 시 네트워크 mocking
- **페이지 초안 구현**: 대시보드, 로그인, 할 일 목록(가상·무한 스크롤), 상세(404·삭제 모달), 회원정보
- **공통 UI**: 모달(ESC·배경 닫기 등)
- **테스트 코드 작성**: Vitest + RTL, 일부 페이지/엔티티 단위 테스트, Playwright E2E 스펙(로그인)
- **문서 초안 작성**

## 3. 핵심 프롬프트 진행 흐름

- 요구사항, openapi 파일을 기준으로 기능 구현 플랜과 TODO를 나누고, 항목별로 순차 구현
  - 구현 → lint, test 확인 → 다음 TODO 진행
- FSD / MSW / TanStack Query / Suspense 등 상황에 따른 기술 선택과 장단점 비교를 정리한 뒤 코드 반영
- 타입 정의, 인증 토큰 정책 등은 대화로 합의 후 반영

## 4. 사람이 최종 검증한 내용

- 로컬 개발환경에서 메인 유저 시나리오를 수동으로 검증하였습니다.

  ### GNB / LNB
  - [ ] 비로그인: 로그인 링크·대시보드·할 일 네비 동작
  - [ ] 로그인 후: 회원정보 진입 링크(또는 동일 역할 UI) 동작

  ### 로그인 (`/sign-in`)
  - [ ] 이메일·비밀번호 라벨 표시
  - [ ] 유효성 메시지 표시
  - [ ] 조건 충족 시에만 제출 활성
  - [ ] 실패 시 `errorMessage` 모달

  ### 대시보드 (`/`)
  - [ ] `numOfTask` / `numOfRestTask` / `numOfDoneTask` 표시

  ### 할 일 목록 (`/task`)
  - [ ] 카드에 `title`·`memo`
  - [ ] 스크롤 시 DOM 부담이 과하지 않음(가상 스크롤)
  - [ ] 목록 하단 근처로 스크롤 이동 시 다음 페이지 로딩(무한 스크롤)
  - [ ] 카드 클릭 시 상세 경로로 이동

  ### 할 일 상세 (`/task/:id`)
  - [ ] 상세 필드 표시(등록일시 포맷 확인)
  - [ ] 존재하지 않는 id: empty state + 목록 복귀
  - [ ] 삭제: 확인 모달, id 일치 시에만 삭제 활성, 성공 후 목록으로 이동

  ### 회원정보 (`/user`)
  - [ ] `name`·`memo` 표시
  - [ ] (구현된 경우) 로그아웃 후 로그인 페이지 등 기대 동작

  ### 인증·세션
  - [ ] 새로고침 후에도(리프레시 쿠키가 있는 경우) 세션 복구 기대 동작
  - [ ] 잘못된 자격으로 로그인 실패 처리
